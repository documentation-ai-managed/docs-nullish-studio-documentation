## Authentication Overview

Authentication in Nullish Studio is designed to be secure, flexible, and easy to implement. We support multiple authentication methods to accommodate different use cases, from simple API key access to advanced OAuth flows for enterprise integrations.

<Callout kind="info">
  All API requests must include valid authentication credentials to access protected resources.
</Callout>

## API Key Authentication

The simplest way to authenticate with Nullish Studio is using API keys. You can generate keys in your account settings and use them for programmatic access.

<Steps>
  <Step title="Generate API Key" icon="key">
    Log into your Nullish Studio account and navigate to the API Keys section in settings.
  </Step>
  <Step title="Configure Permissions" icon="shield">
    Set appropriate permissions for the key based on the operations you need.
  </Step>
  <Step title="Use in Requests" icon="send">
    Include the key in your API requests using the Authorization header.
  </Step>
</Steps>

## OAuth 2.0 Integration

For more advanced scenarios, we support OAuth 2.0 flows, allowing you to authenticate users through external identity providers.

<Tabs>
  <Tab title="Authorization Code" icon="lock">
    The most secure flow for web applications, requiring user consent.
    
    <CodeGroup tabs="JavaScript,Python">
      ```javascript
const authUrl = 'https://nullish.studio/oauth/authorize';
const params = {
  client_id: 'your_client_id',
  redirect_uri: 'https://yourapp.com/callback',
  response_type: 'code',
  scope: 'read:docs write:docs'
};

window.location.href = `${authUrl}?${new URLSearchParams(params)}`;
```
      ```python
import requests
from urllib.parse import urlencode

auth_url = 'https://nullish.studio/oauth/authorize'
params = {
    'client_id': 'your_client_id',
    'redirect_uri': 'https://yourapp.com/callback',
    'response_type': 'code',
    'scope': 'read:docs write:docs'
}

response = requests.get(f'{auth_url}?{urlencode(params)}')
print(response.url)
```
    </CodeGroup>
  </Tab>
  <Tab title="Client Credentials" icon="server">
    For machine-to-machine authentication without user interaction.
    
    <CodeGroup tabs="cURL,Bash">
      ```bash
curl -X POST https://nullish.studio/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET"
```
      ```bash
#!/bin/bash
TOKEN_URL="https://nullish.studio/oauth/token"
CLIENT_ID="YOUR_CLIENT_ID"
CLIENT_SECRET="YOUR_CLIENT_SECRET"

curl -s -X POST $TOKEN_URL \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=$CLIENT_ID&client_secret=$CLIENT_SECRET" | jq -r '.access_token'
```
    </CodeGroup>
  </Tab>
</Tabs>

## Token Management

Access tokens have a limited lifespan for security. Learn how to handle token refresh and storage.

<Expandable title="Token Refresh" default-open="false">
  When your access token expires, use the refresh token to obtain a new one without re-authenticating the user.
  
  ```javascript
const refreshToken = async (refreshToken) => {
  const response = await fetch('https://nullish.studio/oauth/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      grant_type: 'refresh_token',
      refresh_token: refreshToken,
      client_id: 'your_client_id',
      client_secret: 'your_client_secret'
    })
  });
  
  return response.json();
};
```
</Expandable>

<Expandable title="Secure Token Storage" default-open="false">
  Store tokens securely using platform-specific secure storage mechanisms.
  
  For web applications, use HttpOnly cookies. For mobile apps, use Keychain (iOS) or Keystore (Android).
</Expandable>

## Rate Limiting

To ensure fair usage and protect our services, we implement rate limiting on API requests.

| Plan Type | Requests per Hour | Requests per Day |
|-----------|-------------------|------------------|
| Free      | 1,000            | 10,000          |
| Pro       | 10,000           | 100,000         |
| Enterprise| Unlimited        | Unlimited       |

<Callout kind="alert">
  Exceeding rate limits will result in HTTP 429 responses. Implement exponential backoff in your retry logic.
</Callout>

## Best Practices

Follow these security best practices when implementing authentication in your applications.

- Rotate API keys regularly and revoke compromised keys immediately
- Use the principle of least privilege when assigning permissions
- Implement proper error handling for authentication failures
- Monitor authentication logs for suspicious activity
- Use HTTPS for all authentication-related requests

By following these authentication patterns, you'll ensure secure and reliable access to Nullish Studio's powerful documentation platform.
